## 📖 Descripción
Reescribir `database.js` para incluir un sistema de migraciones por versión usando
`PRAGMA user_version`. Cada versión agrega tablas/columnas de forma segura y acumulativa.
Incluye las tablas `settings` (v2) y `users` (v3).

## 🎯 Objetivo
`database.js` tiene la función `runMigrations(db)` que aplica automáticamente
v0→v1 (schema base), v1→v2 (IVA + settings) y v2→v3 (users) sin perder datos.

## ✅ Criterios de Aceptación
- [x] `PRAGMA user_version` llega a `3` después de la primera ejecución
- [x] Al abrir con DB existente v1, solo aplica las migraciones faltantes
- [x] Logs estructurados: `[DB] Migración vX aplicada`
- [x] La tabla `settings` tiene valores por defecto insertados en v2
